# 大心居家護理所
## 系統整合專案提案書（客戶版）

- 提案日期：2026-02-21
- 提案期間：2026-03-01 至 2026-06-30
- 提案對象：大心居家護理所（居家護理 + 長照居服）
- 提案主軸：以最小變動、最低風險完成「三核心系統 + n8n」整合上線

---

## 1. 執行摘要

本提案以「保留既有優勢系統、補上跨系統自動化」為原則，將現行多系統分散作業收斂為：
- 健保居護核心：`neoCare`（保留）
- 長照居服核心：`JUBO`（果雲升級路徑）
- 人事差勤核心：`WorkDo`（替代 MAYO）
- 自動化整合中介：`n8n`（自架、零授權）

### 預期成果（2026-06-30 前）
- 薪資整合流程自動化上線（2026-03-31）
- 每日病歷/照護紀錄備份機制上線（2026-04-30）
- 整體系統整合驗收完成（2026-06-30）

---

## 2. 客戶痛點與目標對應

### 目前痛點
- 同筆資料跨系統重複輸入，行政耗時高
- 打卡、排班、請假、服務紀錄無法自動對帳
- 人員操作介面過多，教育成本高
- 備份與稽核軌跡不足，合規風險存在

### 對應目標
- 系統收斂：7 套降至 3 套核心系統
- 流程自動化：建立 n8n 中介層
- 合規可稽核：每日備份 + 日誌 + 告警
- 管理可視化：固定週期輸出薪資與營運報表

---

## 3. 整合方案設計

### 3.1 目標架構
- `neoCare`：健保申報、病歷、巡診排班（維持主系統）
- `JUBO`：長照排班、照服管理、長照核銷（果雲遷移）
- `WorkDo`：GPS 打卡、請假、核銷、內部溝通
- `n8n`：API/檔案流程編排、排程、通知、告警與資料中轉

### 3.2 關鍵工作流
1. 薪資整合工作流（每月最後工作日）
- 來源：WorkDo 打卡/請假 + neoCare/JUBO 排班
- 核心：自動比對實際時數、套用薪資規則
- 輸出：Excel/CSV + Email 通知人資

2. 每日備份工作流（每日 02:00）
- 來源：neoCare 病歷（PDF）+ JUBO 服務紀錄（JSON）
- 核心：依年/月/個案自動分類、寫入 NAS/雲端
- 控制：失敗告警、備份日誌、抽查機制

## 4. 專案時程與里程碑

### Phase 1：啟動與準備（2026-03-01～2026-03-14）
- MAYO 合約盤點與退場策略確認
- 果雲→JUBO 遷移方案與資料欄位盤點
- WorkDo 帳號建置與教育訓練排程
- n8n 測試環境建置與 API 權限取得

### Phase 2：薪資自動化上線（2026-03-15～2026-03-31）
- 開發 WorkDo + neoCare/JUBO 薪資整合流程
- 進行平行試算與差異修正
- 里程碑 M1：2026-03-31 薪資自動化上線

### Phase 3：備份與驗收（2026-04-01～2026-06-30）
- 每日備份流程上線與告警機制建立
- SOP 文件化、教育訓練、壓力測試
- 里程碑 M2：2026-04-30 每日備份上線
- 里程碑 M3：2026-06-30 全案驗收完成

## 6. KPI 與驗收標準

- 薪資核算人工工時下降 `>= 50%`
- 每日備份成功率 `>= 99%`
- 異常告警觸發時間 `<= 30 分鐘`
- WorkDo 打卡覆蓋率 `>= 95%`
- 每月固定產出薪資報表且可稽核追溯

---

## 7. 角色分工（RACI 摘要）

- 行政主管：A（決策與跨部門協調）
- IT/資訊：R（環境建置、介接、維運）
- n8n 開發：R（流程開發、測試、上線）
- 主任護理師 / 居服督導：C（業務流程驗證）
- 人資顧問：A/C（計薪規則確認與驗收）
- 系統供應商（neoCare/JUBO/WorkDo）：C（API、資料遷移與技術支援）

## 9. 風險控管與保證機制

### 主要風險
- 使用者導入抗拒
- 果雲→JUBO 資料遺失或欄位不一致
- API/憑證異常導致流程中斷
- 病歷保存與稽核規範落差

### 因應措施
- 1 個月導入緩衝與雙軌並行
- 遷移前完整備份、遷移後逐筆抽驗
- 憑證到期檢查 + 失敗告警 + 人工備援流程
- 備份日誌留存與定期稽核檢查表

---

## 10. 前置需求與假設

- 客戶需提供 API 權限或穩定匯出機制（CSV/JSON）
- 客戶需指定專案窗口（行政、護理、居服、IT 各 1 人）
- 合約與資料權責需於 Phase 1 完成確認
- 驗收日以 2026-06-30 為目標，若供應商介接延遲則需同步調整里程碑

---

## 11. 提案結論與建議決策

建議大心採「分階段可驗收」策略：
- 先完成 3 月薪資整合（直接支援人資）
- 再完成 4 月每日備份（強化合規）
- 最後於 6 月完成全案驗收與交接

此路徑可在不重建核心系統的前提下，最快取得可量化效益，且整體風險最低。
